# システムへのアクセス方法

## 🚀 開発サーバーの起動

### 方法1: PowerShellで起動（推奨）

1. PowerShellを開く
2. プロジェクトディレクトリに移動：
   ```powershell
   cd C:\Users\ngf005148\Desktop\webapps\talent
   ```

3. 開発サーバーを起動：
   ```powershell
   npm run dev
   ```

4. 以下のメッセージが表示されるまで待つ：
   ```
   ▲ Next.js 14.0.4
   - Local:        http://localhost:3000
   - ready started server on 0.0.0.0:3000
   ```

5. ブラウザで以下のURLにアクセス：
   - **ホームページ**: http://localhost:3000
   - **ログインページ**: http://localhost:3000/login

## 🔍 アクセスできない場合の確認事項

### 1. サーバーが起動しているか確認

PowerShellで以下のコマンドを実行：
```powershell
netstat -ano | findstr :3000
```

**出力例（正常）:**
```
TCP    0.0.0.0:3000    0.0.0.0:0    LISTENING    12345
```

**何も表示されない場合**: サーバーが起動していません。`npm run dev` を実行してください。

### 2. ポート3000が使用中の場合

他のアプリケーションがポート3000を使用している可能性があります。

**解決方法:**
```powershell
# ポート3000を使用しているプロセスを確認
netstat -ano | findstr :3000

# プロセスIDを確認して終了（例: PID 12345）
taskkill /PID 12345 /F

# 再度サーバーを起動
npm run dev
```

**別のポートを使用する場合:**
```powershell
npm run dev -- -p 3001
```
→ ブラウザで `http://localhost:3001` にアクセス

### 3. ブラウザでアクセス

以下のURLをブラウザのアドレスバーに入力：

- **ホームページ**: `http://localhost:3000`
- **ログインページ**: `http://localhost:3000/login`

**注意**: `https://` ではなく `http://` を使用してください。

### 4. ブラウザのキャッシュをクリア

古いキャッシュが原因の場合があります：
- Chrome: `Ctrl + Shift + Delete` → キャッシュをクリア
- または、シークレットモード（`Ctrl + Shift + N`）で試す

## 📋 必要な設定

### 環境変数の確認

`.env.local` ファイルが存在し、正しく設定されているか確認：

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

**確認方法:**
```powershell
# ファイルの存在確認
Test-Path .env.local

# 内容確認（機密情報は表示されません）
Get-Content .env.local
```

## 🐛 エラーが発生した場合

### よくあるエラーと対処法

#### 1. "Cannot find module"
```powershell
# 依存関係を再インストール
npm install
```

#### 2. "Port 3000 is already in use"
```powershell
# ポート3000を使用しているプロセスを終了（上記参照）
# または別のポートを使用
npm run dev -- -p 3001
```

#### 3. "Missing Supabase environment variables"
→ `.env.local` ファイルを確認し、正しい値を設定してください。

#### 4. ブラウザで「接続できません」と表示される
- サーバーが起動しているか確認
- ファイアウォールの設定を確認
- ブラウザのキャッシュをクリア
